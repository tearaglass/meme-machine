Project 002 - Meme Machine

Version: v1.0

Status: Authoritative

0. Purpose

This system is a Telegram-native meme creation and routing tool composed of:

  • a Telegram Bot (control, routing, posting)
  • a Telegram Mini App (meme editor)

The system is optimized for:

  • speed
  • repeatability
  • aesthetic control
  • distribution inside Telegram

It is not intended to be a general design tool.

1. Hard Constraints

These rules are non-negotiable.

  • Telegram is the only required platform
  • No external authentication
  • Mini App creates memes only
  • Bot posts memes only
  • No React unless unavoidable
  • No feature creep outside defined scope
  • Editor logic must be deterministic and debuggable

2. System Overview

Components

  1. Telegram Bot
    • Entry point
    • Command handling
    • Session tracking
    • Meme posting
  1. Telegram Mini App
    • Web-based canvas editor
    • Runs inside Telegram WebApp container
    • Returns final image to bot
  1. Asset Stores
    • Base Image Bank
    • Sticker Library
    • Template Library
    • Font Library

High-Level Flow

User
 → Telegram Bot Command
 → Telegram Mini App
 → Meme Creation
 → Export Image
 → Bot Receives Image
 → Bot Posts to Chat(s)

3. Telegram Bot Specification

Responsibilities

The bot is responsible for:

  • Launching the mini app
  • Maintaining lightweight user sessions
  • Receiving exported meme images
  • Posting memes to Telegram chats or channels
  • Storing minimal temporary metadata

Responsibilities Explicitly Excluded

The bot must not:

  • Render images
  • Edit images
  • Modify canvas content
  • Understand layers or templates

Required Bot Commands

/start        – introduction and usage
/meme         – launch meme editor mini app
/templates    – list available templates
/stickers     – list sticker packs
/history      – last N memes (session scoped)
/send         – send last meme to selected chats
/settings     – user defaults (future use)

Bot State Model

Bot state is ephemeral by default.

{
  "userId": "telegram-id",
  "activeSession": true,
  "lastMemeFileId": "telegram-file-id | null",
  "pendingDestinations": []
}

4. Telegram Mini App Specification

Responsibilities

The mini app is responsible for:

  • Rendering the meme editor UI
  • Managing canvas state
  • Managing layers
  • Exporting the final image

Environment

  • Runs as a Telegram Web App
  • Receives user + session context from Telegram
  • No accounts
  • No persistence required for V1

The UI is divided into functional zones, not aesthetic prescriptions.

Zones

  1. Canvas Area
    • Primary editing surface

6. Asset Systems

6.1 Base Image Bank (Background Images)

Purpose

A curated, predefined set of images users can start from without uploading their own.

Base images:

  • establish house style
  • speed up meme creation
  • are read-only assets

Base Image Object

{
  "id": "base-image-id",
  "name": "string",
  "src": "url-or-local-path",
  "defaultCanvas": {
    "width": 1080,
    "height": 1080
  },
  "tags": ["reaction", "format"],
  "credit": "optional string"
}

Behavior Rules

  • Selecting a base image initializes the canvas
  • Base image is inserted as the bottom-most layer
  • Base image layer is locked
  • Locked layers:
    • cannot be deleted
    • cannot be reordered
    • may be scaled or repositioned

6.2 Templates

Templates define entire canvas states, not just backgrounds.

Template Object

{
  "id": "template-id",
  "name": "string",
  "canvas": {
    "width": 1080,
    "height": 1080
  },
  "layers": []
}

Templates may include:

  • base images
  • text layers
  • sticker layers

Templates are optional in V1 but supported.

6.3 Stickers

  • Transparent PNGs
  • Drag-and-drop overlays
  • Stored locally or via static URLs
  • No user uploads in V1

6.4 Fonts

  • Local web fonts
  • Loaded at startup
  • Small, curated set in V1

7. Canvas & Layer Model

Layer Object (Generic)

{

9. Export & Handoff Protocol

Export Rules

  • Format: PNG
  • Resolution: canvas native resolution
  • Background:
    • transparent if no base image
    • opaque if base image present

Handoff Process

  1. Mini app exports canvas to image blob
  1. Mini app sends blob via Telegram WebApp API
  1. Bot receives file
  1. Bot stores temporarily
  1. Bot asks user for destination

10. Message Routing

Supported Destinations

  • Current chat
  • Selected chats
  • Channels where bot has permission

Routing Flow

Receive Image
 → Preview
 → User selects destinations
 → Bot posts image
 → Session resets or continues

11. Error Handling

Mini App

  • Prevent export if canvas is empty
  • Warn before discarding unsaved work
  • Gracefully handle asset load failures

Bot

  • Handle permission errors
  • Handle upload failures
  • Retry once, then fail clearly
